Guía Angular - Vercel

Instrucciones paso a paso para desplegar tu aplicación Angular en Vercel

1

Preparar tu build Angular

Primero, asegurate de tener el build de producción listo:

ng build --configuration production

Esto va a generar la carpeta dist/angular_app_2/browser (o como se llame tu outputPath).

2

Subir tu proyecto a GitHub / GitLab / Bitbucket

Vercel se conecta automáticamente con tus repos.

Ya que vos ya tenés el proyecto en GitHub, ¡perfecto!

3

Ir a vercel.com

Iniciá sesión con tu cuenta de GitHub.

Dale a "New Project".

Elegí tu repo angular_app_2.

4

Configurar el proyecto Angular

Cuando te pregunte por los comandos de build, completá así:

Framework Preset
Other (no elijas Angular directamente)
Build Command
ng build --configuration production
Output Directory
dist/angular_app_2/browser
Install Command
npm install (o dejalo en blanco si querés que lo detecte)
5

Desplegar

Haz clic en "Deploy" y espera a que Vercel complete el proceso de despliegue.

Una vez finalizado, Vercel te proporcionará una URL para acceder a tu aplicación desplegada.

6

Configurar dominio personalizado (opcional)

Si deseas usar un dominio personalizado:

  1. Ve a la sección "Domains" en el panel de control de tu proyecto
  2. Agrega tu dominio personalizado
  3. Sigue las instrucciones para configurar los registros DNS
+

Ventajas de usar Vercel

Vercel ofrece varias ventajas para el despliegue de aplicaciones Angular:

Despliegue continuo

Cada vez que haces push a tu repositorio, Vercel automáticamente despliega los cambios.

Vista previa por rama

Cada pull request genera un despliegue de vista previa para facilitar las pruebas.

Certificados SSL automáticos

Vercel proporciona certificados SSL gratuitos para todos los sitios, incluso con dominios personalizados.

Red global CDN

Tu aplicación se sirve desde ubicaciones de borde en todo el mundo para una carga más rápida.
