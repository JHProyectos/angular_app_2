Introducción

1

API First

Es el desarrollo de una API como primera prioridad porque es más ordenado primero establecer los contratos del código y luego realizar el código

2

Historias de Usuario

Super importante primero definir las Historias de Usuario antes de arrancar

Es importante escribir las historias de usuario para que sólo hagan referencia a un Verbo HTTP

3

Desarrollo

La aplicación se desarrolla según el sistema, no al revés

4

Contrato

Es la especificación de la API (Endpoints, métodos, datos, request, response)

Define:

Métodos HTTP soportados

los Endpoints

El cuerpo de la solicitud (request body)

Respuestas(responses)

Los esquemas de datos (data schemas)

Mecanismos de autenticación y autorización

posibles limitaciones (rate limiting)
